## 민감정보 제외 기준

이 문서는 인수인계와 점검용 요약입니다. 다음 정보는 의도적으로 제외했습니다.

- API 키, 인증 토큰, 계좌 관련 정보
- 실제 보유 종목별 상세 수량, 평가액, 손익 금액
- 사용자가 입력한 원문 전문, 첨부 파일 원본 내용
- 유료/프리미엄 콘텐츠 URL 전문과 본문 전문
- 브라우저 세션/개인 환경에만 필요한 임시 값

## 작업 원칙

- 투자 리서치 OS 작업은 `C:\Users\lib20\InvestmentJournalApp`에서만 진행했습니다.
- OneDrive 경로는 작업 루트로 사용하지 않았습니다.
- 화면 결과와 사용자 메시지는 한국어 중심으로 유지했습니다.
- 사용자가 버튼 클릭 후 진행 여부를 알 수 있도록 화면 피드백을 강화했습니다.

## 주요 완료 작업

### 1. 정보 입력 저장 후 초기화 및 화면 피드백

- 정보 입력 저장 성공 후 입력 화면이 정리되도록 기존 흐름을 유지 및 검증했습니다.
- 주요 버튼 클릭 시 상단 알림 바에 `요청 접수` 메시지가 즉시 표시되도록 개선했습니다.
- 적용 범위:
  - 대시보드 조회
  - 최신 데이터 조회
  - 리포트 실행
  - 티커 진단
  - 차트 분석 이동
  - 정보 입력 저장 및 본문 미리보기
  - 저장 데이터 조회
  - 포트폴리오 불러오기/저장/삭제/기간 수익 비교
  - 관심종목/섹터 추가/저장/불러오기

### 2. 해외 URL 및 웹 본문 처리

- 해외 URL 입력 시 한국어 분석용 요약/변환 결과가 화면에 보이는 흐름을 검증했습니다.
- 웹사이트 본문 미리보기에서 해외 문서의 처리 상태, 언어, 한국어 변환 상태가 표시되도록 확인했습니다.
- 웹사이트에 접속은 되지만 본문 추출이 충분하지 않은 경우, 오류로 중단하지 않고 URL-only 자료로 저장하도록 개선했습니다.
- 이 경우 저장 데이터에는 링크, 제목, 처리 상태, 보강 필요 안내가 남습니다.
- `needs_body_copy`, `url_text_unavailable` 태그로 추후 본문 보강이 필요한 자료임을 표시합니다.

### 3. 관심종목/섹터 UI 정리

- 관심종목은 회사명 중심으로 보이도록 정리했습니다.
- 대시보드의 관심종목 빠른 선택 칩에서 종목코드 표시를 제거했습니다.
- 관심종목/섹터 탭의 관심종목 카드에서도 회사명 아래 티커 코드 줄을 제거했습니다.
- 내부 선택값과 실행용 티커 데이터는 유지되어, 화면만 단순해졌습니다.

### 4. 저장 데이터 화면 정리

- 저장 데이터 조회 시 공식 인증 파일을 먼저 보여주도록 구성했습니다.
- 레거시/검증 전 파일은 접힌 목록으로 분리했습니다.
- 저장 데이터 경고도 접힘 영역으로 정리해 화면 점유를 줄였습니다.
- 이미지/OCR 또는 첨부 파일이 있는 항목은 카드에서 처리 상태와 추출 본문 길이를 확인할 수 있게 했습니다.

### 5. 이미지/OCR 및 파일 추출 상태 표시

- 이미지 파일 업로드 후 OCR 상태를 더 명확하게 표시하도록 개선했습니다.
- 표시 예:
  - OCR/추출 완료
  - OCR 본문 없음
  - OCR 오류
  - OCR 미연결
  - 본문 추출 글자 수
- 파일 처리 리포트에도 OCR 상태와 추출 품질, 다음 조치가 표시되도록 보강했습니다.

### 6. 포트폴리오 조회 및 기간 수익 비교

- 포트폴리오 조회 화면에 최근 불러온 일시가 표시되도록 했습니다.
- 최근 1주일, 1개월, 6개월, 1년 기준 기간 수익 비교 기능을 추가했습니다.
- 화면에는 다음 정보가 카드 형태로 표시됩니다.
  - 현재 미실현 손익 요약
  - 현재 미실현 수익률 요약
  - 기간별 순수익
  - 기간별 수익률
  - 가격 기준일
  - 포함 종목 수
  - 제외 종목 접기/펼치기
- 실제 보유 종목별 수량과 상세 손익 수치는 이 요약 문서에서 제외했습니다.

### 7. 저장 데이터 조회 오류 개선

- 저장 데이터 조회 버튼 클릭 시 결과가 너무 빨리 바뀌어 메시지가 안 보이는 문제를 해결했습니다.
- 결과창과 독립된 상단 알림 바를 추가했습니다.
- 저장 데이터 조회, 기간 수익 비교, 정보 입력 저장 등 주요 액션에서 즉시 피드백이 보입니다.

### 8. 네이버 프리미엄/본문 추출 제한 URL 처리

- 일부 웹사이트는 백엔드가 접속해도 본문 텍스트를 충분히 추출하지 못합니다.
- 기존에는 이 경우 HTTP 422 오류로 저장이 중단되었습니다.
- 현재는 오류로 중단하지 않고 URL 보관 자료로 저장합니다.
- 이후 사용자가 본문을 직접 복사하거나 파일을 첨부해 보강할 수 있습니다.
