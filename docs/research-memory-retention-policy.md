# Research Memory Retention Policy

## 원칙

- 저장 리포트는 기본적으로 하드 삭제하지 않는다.
- 공식 인증 리포트가 투자 판단의 우선 자료다.
- 레거시/검증 전 파일은 원문 추적과 감사 목적의 보관 자료로 취급한다.
- 레거시 파일 정리는 `status=archived`, `is_deleted=true` 메타데이터를 기록하는 소프트 보관으로 처리한다.

## 화면 동작

- 기본 저장 데이터 목록은 공식 인증 파일을 먼저 보여준다.
- 레거시 파일은 접힌 그룹으로 분리한다.
- 보관 처리된 문서는 기본 목록과 자동 주입 후보에서 숨긴다.
- `보관 문서 포함` 옵션을 켜면 보관 문서를 다시 확인하고 개별 복원할 수 있다.

## 일괄 보관

- `레거시 일괄 보관`은 활성 레거시 파일만 대상으로 한다.
- 파일 본문과 JSON sidecar는 삭제하지 않는다.
- manifest와 JSON sidecar에 보관 상태, 보관 시각, 보관 사유를 남긴다.
- 복원은 개별 파일 단위로 수행한다.

## 금지

- 레거시 파일을 자동으로 물리 삭제하지 않는다.
- 공식 인증 파일이 없는데 레거시 전체를 판단 자료에서 완전히 제거하지 않는다.
- 보관 문서를 사용자가 볼 수 없는 상태로 만들지 않는다.
